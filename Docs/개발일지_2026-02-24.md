### 기술 메모

- **Registry 기반 이벤트**: 대화 종료·퀘스트 완료를 각각 별도 이벤트로 분리. 핸들러가 인터페이스 구현 후 Register, 발행 시 InvokeAll로 전달. Controller는 라우터만 담당, Registry에 등록하지 않음.

- **Presenter Request 패턴**: Dialogue와 동일하게 Quest도 Controller → Presenter.RequestXxx → System 호출. Accept/Complete 로직을 Presenter에 위임해 일관된 구조 유지.

- **퀘스트 완료 처리 분리**: CompleteQuest는 QuestSystem에서만 호출, OnQuestCompleted 발행. 플래그 설정·인벤토리 차감은 IQuestCompletedHandler(QuestCompletedFlagHandler, QuestInventoryHandler)가 담당.

- **플래그 핸들러 분리**: DialogueFlagHandler(대화 종료 시 flagsToModify), QuestCompletedFlagHandler(퀘스트 완료 시 quest_completed)를 이벤트 타입별로 분리. QuestInventoryHandler는 도메인 결합 방지를 위해 별도 유지.

- **QuestFlagSync**: OnQuestUpdated 구독, 목표 달성 시 QuestObjectivesDone 플래그 설정. 대화 선택(QuestComplete)에서 "목표 달성" 여부 확인용.
