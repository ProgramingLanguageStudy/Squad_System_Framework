# DialogueUI 역할 정의

DialogueUI가 담당하는 범위를 한 줄로 정리한 문서.

---

## 역할 (책임)

1. **필요한 정보만 받아서** 대화창과 버튼을 **띄운다**.
2. **부가적으로** “다음 대사로 넘기기”(다음 문장 표시)를 처리한다.

**끝.**  
버튼을 눌렀을 때 **무슨 일이 일어나는지**(조건 체크, 인벤토리 열기, 퀘스트 수락, 호감도 선물 등)는 **알 바가 아니다**.  
클릭 시 “이 버튼이 눌렸다”만 알리고(이벤트/콜백), 실제 동작은 구독하는 쪽에서 처리.

---

## 담당하지 않는 것

- 버튼 클릭 후 비즈니스 로직 (퀘스트 수락/제출, 선물, 플래그 설정 등)
- 조건 판단 (지금 퀘스트 버튼을 보여줄 수 있는지 등) → 이건 대화를 *시작*하기 전에, 대화를 여는 쪽에서 판단해서 “버튼 정보 목록”으로 넘겨 줌
- 인벤토리/다른 UI 열기 등 연쇄 동작

---

## 한 줄 요약

**“받은 정보로 대화창·버튼을 그리고, 다음 문장 넘기기 + 버튼 클릭은 알리기만 한다.”**

이 정도로 역할을 고정해 두면 단일 책임이 잘 지켜진다.
