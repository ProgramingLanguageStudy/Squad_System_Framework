# 게임 확장 계획 정리

> 현재 코드베이스(퀘스트, 인벤토리, 스쿼드, 캐릭터, 세이브 등)를 기반으로 확장 아이디어를 정리한 문서입니다.

---

## 1. 캐릭터 성장 시스템

### 1.1 레벨 개념
- **경험치 재료 방식**: 사냥 경험치가 아닌, **경험치 아이템(소모)**으로 레벨업
- `ItemType.Material` 계열 또는 별도 `ItemType.ExpMaterial` 추가 가능
- 필요 수량: 레벨별 테이블 (예: Lv2 = 1개, Lv3 = 2개, Lv4 = 4개...)

### 1.2 스탯 명확화
- **CharacterData**: 초기/디자인 타임 스탯 (maxHp, moveSpeed, attackPower, attackSpeed, defense)
- **CharacterModel**: 런타임 최종 스탯 = BaseStats + 레벨 보너스 + 장비 보정 + 버프
- 추가 고려 스탯: 크리티컬, 회피, 스킬 쿨타임 감소 등

### 1.3 레벨 연동
- `CharacterMemberData` (세이브)에 `level` 필드 추가
- 레벨업 시 StatModifier 형태로 보정 적용
- 최대 레벨, 레벨당 스탯 증가량 정의 필요

## 2. 재화 시스템

### 2.1 골드
- `QuestData`에 이미 `RewardGold` 존재
- **별도 골드 시스템** 필요: Wallet / CurrencyManager 등
- 인벤토리와 독립, 세이브/로드에 포함
- UI: HUD에 골드 표시, 상점·강화 등에서 사용

### 2.2 기타 재화 (선택)
- 보석, 영혼석 등 추가 재화 타입
- 퀘스트·일일 보상·이벤트 등에서 지급

## 3. 캐릭터 기능 추가

### 3.1 대시(Dash)
- `CharacterStateMachine`에 `DashState` 추가
- 쿨다운 적용, 무적 프레임 여부 결정
- InputHandler에 대시 입력 매핑

### 3.2 스킬
- **스킬 데이터**: ScriptableObject (이름, 쿨다운, 마나/스태미나 소모, 효과 등)
- **스킬 타입**: 즉발 데미지, 넉백, AoE, 버프 등
- CharacterModel 또는 별도 SkillController
- UI: 스킬 바, 쿨다운 표시

### 3.3 기타 기능
- 점프(3D일 경우), 파리기, 블록/가드 등

## 4. 장비 아이템

### 4.1 현재 상태
- `EquipmentItemData` 존재 (StatModifier 포함)
- `ItemType.Equipment` 정의됨
- 실제 **착용/해제 로직·UI 미구현**

### 4.2 처리 방안

| 항목 | 설명 |
|------|------|
| **장비 슬롯** | 무기, 방어구, 악세서리 등 슬롯 타입 정의 |
| **착용 UI** | 인벤토리 아이템 드래그→장비 슬롯 또는 더블클릭 |
| **CharacterModel 연동** | 장비 착용 시 AddModifier, 해제 시 SubtractModifier |
| **세이브** | 캐릭터별 착용 장비 ID 저장 |
| **인벤토리** | 장비는 인벤토리에 있고, "착용 중" 마킹 or 장비 전용 슬롯 |

### 4.3 권장 구현 순서
1. `EquipmentSlot` enum (Weapon, Armor, Accessory 등)
2. `CharacterEquipment` (또는 CharacterModel 내부) — 착용 중인 장비 목록
3. 인벤토리 `TryEquipItem` → CharacterEquipment에 등록 + Modifier 적용
4. 장비 UI 창

## 5. UI 확장

### 5.1 퀘스트
| 구분 | 설명 | 현재 |
|------|------|------|
| **퀘스트 트래커** | 진행 중 퀘스트 요약 (HUD) | ✅ QuestView |
| **퀘스트 목록 창** | 수락/진행/완료 퀘스트 전체, 수락/완료 버튼 | ❌ 미구현 |

### 5.2 플레이 화면
| UI | 설명 | 현재 |
|----|------|------|
| **현재 조종 캐릭터 체력** | 체력바 | ✅ PlaySceneView |
| **분대원 상태 아이콘** | 각 캐릭터 HP/상태 표시 (소형 아이콘·바) | ❌ 미구현 |
| **스킬 바** | 스킬 버튼 + 쿨다운 | ❌ |
| **골드/재화** | HUD 표시 | ❌ |

### 5.3 창(Window) UI
| 창 | 설명 |
|----|------|
| **캐릭터 상태창** | 레벨, 스탯, 장비, 스킬 등 종합 |
| **퀘스트 목록** | 위와 동일 |
| **인벤토리** | ✅ 있음, 장비 착용 연동 필요 |
| **설정** | 사운드, 키바인딩 등 |
| **세이브/로드** | 저장 슬롯 선택 UI (선택) |

### 5.4 공통
- **툴팁**: 아이템, 스킬 호버 시 설명
- **팝업/모달**: 확인, 알림 창

## 6. 추가로 고려할 수 있는 것들

### 6.1 게임플레이
- **크래프팅**: Material 아이템 → 새 아이템 제작
- **캐릭터 영입**: 퀘스트·이벤트로 동료 추가 (SquadController.AddCompanion 이미 존재)
- **죽음/부활 플로우**: 리스폰 위치, 페널티 등
- **난이도 설정**: 스탯 배율, 드롭률 등

### 6.2 콘텐츠
- **일일/주간 퀘스트**: 보상 강화
- **업적/마일스톤**: 플래그 기반 (QuestController 플래그와 유사)
- **스토리 진행도**: 챕터/플래그로 게임 진행 제어

### 6.3 UX
- **빠른 이동**: 포탈 외 추가 (맵 클릭 등)
- **튜토리얼**: 첫 플레이 가이드
- **일시정지 메뉴**: 설정, 세이브, 메인으로

### 6.4 기술/운영
- **세이브 슬롯**: 여러 세이브 파일 지원
- **설정 저장**: 해상도, 볼륨, 키 바인딩
- **로컬화**: 다국어 (선택)

## 8. 현재 코드와의 연계 요약

| 기존 시스템 | 확장 시 참고 |
|-------------|--------------|
| `CharacterModel` (StatModifier) | 레벨 보너스, 장비 보정 모두 Modifier로 통합 가능 |
| `CharacterData` | 레벨당 스탯 테이블은 별도 SO 또는 DataManager |
| `Inventory` (OnItemChangedWithId) | 경험치 재료 소모 시 퀘스트·레벨업 로직에서 구독 |
| `QuestData` (RewardGold, RewardItems) | 골드·아이템 지급은 QuestController.CompleteQuest 처리 |
| `EquipmentItemData` | StatModifier 필드 이미 있음 → 착용 시 Model.AddModifier |
| `SquadController` (OnPlayerChanged) | 분대원 UI는 이벤트로 현재 분대 목록 구독 |
| `PlaySaveCoordinator` | 골드, 레벨, 장비 착용 상태 등 세이브 포맷 확장 |
