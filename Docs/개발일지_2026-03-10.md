### 기술 메모

- **IntroScene/View 분리**: Presenter-View 패턴 적용. UI 참조·갱신은 View, 비즈니스 로직은 IntroScene.

- **DataManager CacheByType**: `so is CharacterData` 등 런타임 타입 검사로 분류. SO의 `characterId`, `ItemId` 등 ID 필드로 캐시 키 사용.

- **Addressables 전환**: Resources.Load 대신 Addressables 사용. SO는 02_Scripts/0_ScriptableObjects에 두고 Data 라벨로 일괄 로드.

- **Squad 리팩터**: Squad(상태) + PooledSquadCharacterFactory(생성). ResourceManager는 Addressables 기반 GetPrefab, PreloadByLabelAsync.

- **Firebase SDK**: FirebaseAuth, FirebaseFirestore. google-services.json(Android), google-services-desktop.json(PC) 포함. x86_64 FirebaseCppApp-13_9_0.bundle(190MB), .so(122MB)는 GitHub 100MB 제한으로 .gitignore 제외. Windows는 .dll만 사용.

### 배운 점

- Firebase Google Sign-In: Unity Editor에서는 네이티브 플러그인(Play Services 등) 미지원으로 테스트 불가. 모바일 실기기/에뮬레이터 필요.
- PC 빌드: 브라우저 OAuth 플로우로 Google 로그인 가능. Firebase Unity SDK는 PC용 별도 구현 필요.
- 이메일/비밀번호 Auth: Editor 포함 모든 플랫폼에서 동일하게 동작.
