# 세이브 데이터 서버 연동 설계

> Firestore 기반 클라우드 전용 세이브. 로컬 저장 없음.

---

## 1. 최종 결정 사항

| 항목 | 결정 |
|------|------|
| **저장소** | Firestore (`users/{userId}/save/slot_0`) |
| **로컬** | 사용 안 함 |
| **인터페이스** | 사용 안 함 (FirestoreSaveBackend 직접 사용) |
| **게스트** | 없음 (로그인 필수) |
| **타임스탬프/동시 수정** | 불필요 (단일 기기, Last-Write-Wins) |

---

## 2. 저장 시점

| 시점 | 동작 |
|------|------|
| **앱 종료** | `Application.wantsToQuit`으로 저장 완료 대기 후 종료 (빌드) |
| **에디터 플레이 종료** | 베스트 에포트 (완료 보장 어려움) |
| **5분 주기** | 보험용 자동 저장 |
| **씬 전환 (Play→Intro)** | 저장 후 씬 로드 |

---

## 3. 로드 시점

| 시점 | 동작 |
|------|------|
| **Play 진입** | 서버에서 비동기 로드 → 세이브 없으면 새 게임(기본 스폰) |
| **로그인 필수** | `FirebaseAuth.CurrentUser` 없으면 로드 불가, 새 게임 |

---

## 4. 에러 처리

| 상황 | 동작 |
|------|------|
| **로드 실패** | 재시도 → 실패 시 Intro 씬으로 |
| **저장 실패** | 재시도, 실패 알림 |
| **로그인 만료** | 재시도 후 실패 시 Intro 씬으로 |

---

## 5. 아키텍처

```
SaveManager
    ├─ GatherSaveData() / ApplySaveData()  ← 기존 유지
    └─ FirestoreSaveBackend (직접 참조)
           → users/{userId}/save/save_slot0
```

- `SaveAsync()`, `LoadAsync()` 비동기 API
- 문서 구조: `{ "data": "JSON string", "updatedAt": Timestamp }`

---

## 6. Firestore Security Rules

```
match /users/{userId} {
  allow read, write: if request.auth != null && request.auth.uid == userId;
}
```

---

## 7. 구현 순서

1. ~~FirestoreSaveBackend 클래스~~ ✓
2. ~~SaveManager 수정 (Firestore 연동, wantsToQuit, 5분 주기)~~ ✓
3. ~~PlayScene 비동기 로드 (코루틴)~~ ✓
4. ~~SettingsView 저장 후 씬 전환 (SaveAsync → LoadScene)~~ ✓

---

## 8. Firestore 설정

1. Firebase Console → Firestore Database 생성
2. Rules 탭에 아래 규칙 적용:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```
