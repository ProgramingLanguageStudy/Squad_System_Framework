### 기술 메모

- **PlaySceneEventHub**: 플레이 씬 한정 이벤트(OnNpcInteracted, OnEnemyKilled)는 GameEvents 대신 PlaySceneEventHub 사용. 씬 언로드 시 Clear()로 구독 정리.

- **DamageableDetector OnTriggerStay**: Unity에서 트리거가 활성화될 때 이미 겹쳐 있는 오브젝트에는 OnTriggerEnter가 호출되지 않음. OnTriggerStay로 보완해 히트박스 켜진 동안 매 프레임 감지. CharacterAttacker._hitThisAttack으로 중복 타격 방지.

- **퀘스트 TargetId 매칭**: EnemyData.enemyId를 퀘스트 TargetId와 매칭. 처치 퀘스트는 enemyId로 NotifyProgress 호출 시 진행.

- **SaveContributor 패턴**: Gather(저장 시 데이터 수집) → Apply(로드 시 데이터 적용). SaveOrder로 실행 순서 제어. Inventory가 Quest보다 먼저 Apply되어 Gather 퀘스트 진행도 동기화.

### 배운 점

- **Unity OnTriggerEnter 한계**: 콜라이더가 enabled될 때 이미 겹쳐 있는 오브젝트는 OnTriggerEnter가 호출되지 않음. OnTriggerStay 또는 Enable 시점에 Physics.OverlapBox 등으로 수동 검사 필요.

- **Awake/Start 순서**: Inventory.Initialize가 Awake에서 호출되면, PlayScene.Awake에서 Apply를 호출할 때 Awake 순서에 따라 Inventory._slots가 null일 수 있음. Apply는 Start로 옮기면 해결.

- **플레이 씬 이벤트 분리**: 전역 GameEvents와 씬 한정 PlaySceneEventHub를 구분해, 씬 전환 시 이벤트 구독 누수 방지.
